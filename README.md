# GenAIPros.com — Topic-Driven Teaching Engine

**Kamaljeet S Kharbanda** · AI Advisor & Instructor @ AIMLEngineers.com · Building GenAIPros

A Claude Code project that turns my topic notes into three published-ready artifacts: a deep teaching deck (18 slides), a LinkedIn post, and a long-form YouTube script. Topic-driven, not plan-driven. Manual publish.

---

## One-time setup

### 1. Install Claude Code plugins

Inside Claude Code:

```
/plugin marketplace add anthropics/skills
```

Wait for it to confirm. Then on a new line:

```
/plugin install document-skills@anthropic-agent-skills
```

Quit Claude Code (`Ctrl+D`). You'll restart it later.

### 2. Install render and build tools

In your shell:

```bash
brew install --cask libreoffice
brew install poppler
npm install -g pptxgenjs
pip install Pillow
```

### 3. Make scripts executable

```bash
cd GenAIPros.com
chmod +x scripts/*.sh .claude/hooks/*.sh
```

### 4. Initialize git (optional but recommended)

```bash
git init
git add .
git commit -m "Initial commit"
```

## Adding a new topic

There are three paths depending on whether you already have notes.

### Path A — I already have notes in a markdown file

This is the most common path.

```bash
cd GenAIPros.com
claude
```

Inside Claude Code:

```
/start-topic
```

You'll be asked three things:

| Question | Example answer |
|---|---|
| **Topic name** | "Postgres on GCE for AI workloads" |
| **Headline concept** (gets purple accent) | "Postgres on GCE" |
| **Notes source** | `~/Desktop/my-postgres-notes.md` |

`/start-topic` will:

1. Count existing topics, auto-number the new one (Topic 02, 03, etc.)
2. Slugify the topic name (`postgres-on-gce-for-ai-workloads`)
3. Create the folder `topics/02-postgres-on-gce-for-ai-workloads/`
4. Copy your markdown file into `notes.md`
5. Fill in `topic.yaml` with all metadata

Then generate everything:

```
/finish-topic
```

Pick one of the 3 angles `angle-finder` proposes. Wait ~10 minutes. Three files land in `output/t<NN>/`:

- `deck.pptx` — 18-slide branded deck
- `linkedin.md` — 1000-1500 char post in your voice
- `youtube.md` — 5-7 min video script with 60s executive summary

### Path B — I want to study and write notes inside Claude Code

```
/start-topic
```

For "Notes source," answer **from scratch**. The skill creates an empty `notes.md` with a template.

Now open `topics/<NN>-<slug>/notes.md` and write your notes — headers, prose, lists, code blocks, anything. Or ask Claude in the same session to help structure your notes as you think.

When notes are substantial (300+ chars minimum, ideally 1000+):

```
/finish-topic
```

### Path C — Paste notes inline

If your notes are short or you copied them from elsewhere:

```
/start-topic
```

For "Notes source," answer **paste inline**, then paste the content into the next message. The skill writes it to `notes.md` verbatim.

## Revising an already-published topic

Edit `topics/<NN>-<slug>/notes.md` with new content. Then run **one** of:

| Command | What it regenerates |
|---|---|
| `/build-deck` | Just the deck (re-renders deck-outline.md → deck.pptx → QA) |
| `/build-post` | Just the LinkedIn post |
| `/build-video-script` | Just the YouTube script |
| `/finish-topic` | The whole pipeline — you'll re-pick an angle |

For voice-only revisions to a post:

```
/critique
```

Runs the post-critic subagent and surfaces the top 3 suggestions without rewriting.

## Files you touch vs files the system generates

This is the most important table to understand.

| You touch | Generated by skills (don't edit by hand) |
|---|---|
| `topics/<NN>-<slug>/notes.md` | `topics/<NN>-<slug>/deck-outline.md` |
| `topics/<NN>-<slug>/topic.yaml` (mostly auto-filled, you can refine `angle:`) | `output/t<NN>/deck.pptx` |
| `brand/design-config.json` (for re-skinning, rare) | `output/t<NN>/linkedin.md` |
| | `output/t<NN>/youtube.md` |
| | `output/t<NN>/qa/slide-*.jpg` |

**Rule of thumb:** if `/finish-topic` produces something wrong, the fix is almost always in `notes.md` (make the input richer) or `topic.yaml` (refine the chosen angle and re-run). Don't edit `deck-outline.md` directly — it'll get overwritten the next time you run `/finish-topic`.

## What runs when you say `/finish-topic`

You only ever push two commands (`/start-topic`, `/finish-topic`) and answer one question (which angle). Everything else is automation. Here's the sequence:

```
/finish-topic
    ↓
1. Loads context: brand guide, voice anchor, design config, your notes
    ↓
2. angle-finder subagent (~20s, isolated context)
   Reads notes.md, proposes 3 distinct angles
    ↓
3. YOU PICK an angle  ← only human decision
    ↓
4. deck-architect subagent (~1-2 min, isolated context)
   Designs 18 slides matching the design system
   Writes deck-outline.md
    ↓
5. build.js runs (~3s)
   Reads deck-outline.md + design-config.json
   Emits deck.pptx with author footer on every slide
    ↓
6. qa-render.sh runs (~10-30s)
   pptx → PDF → JPGs at 100 DPI
    ↓
7. deck-reviewer subagent (~1-2 min, looks at every JPG)
   Reports defects: overflow, overlap, low contrast, etc.
    ↓
8. Main agent fixes any defects, re-renders (one cycle only)
    ↓
9. linkedin.md drafted (~1 min)
   Following brand/linkedin-style.md exactly
    ↓
10. youtube.md drafted (~2 min)
    5-7 min script with punchy 60s executive summary
    ↓
11. post-critic + script-coach subagents (~1 min combined)
    Voice + spoken-flow checks
    ↓
12. Apply top suggestions inline (one critique cycle)
    ↓
13. Mark status: done in topic.yaml
    ↓
14. Done. Review, edit, publish manually.
```

Total: ~10 minutes for a clean run.

## Design system

Loaded from `brand/design-config.json`. Edit that file to re-skin every future deck.

| Color | Hex | Use |
|---|---|---|
| Slate 900 | `#0F172A` | Primary text, dark slides |
| Slate 50 | `#F8FAFC` | Card backgrounds |
| Slate 200 | `#E2E8F0` | Card borders |
| Sky | `#0284C7` | Category 1 / baseline concept |
| Teal | `#0D9488` | Category 2 / supporting concept |
| Amber | `#D97706` | Category 3 / alternative / warning |
| **Purple** | `#7C3AED` | **Reserved for the headline concept** |

The `headline_concept` field in each `topic.yaml` is the one that gets purple. For Topic 01 it's "MCP". For your next topic it might be "vLLM" or "Postgres on GCE" — whatever the post is making the case for.

Each accent has a soft tint companion (`#E0F2FE`, `#CCFBF1`, `#FEF3C7`, `#F3E8FF`) used for card fills.

## Subagent reference

Subagents run in **isolated context windows** — they read what they need, do their job, and return a summary. Your main thread doesn't burn tokens on their work.

| Agent | Role | Approximate cost |
|---|---|---|
| `angle-finder` | Proposes 3 angles for the post from notes | ~15k tokens, 20s |
| `deck-architect` | Designs 18-slide structure following design system | ~30k tokens, 1-2m |
| `deck-reviewer` | Visually inspects 18 rendered JPGs | ~35k tokens, 1m |
| `post-critic` | Reviews LinkedIn draft against voice anchor | ~10k tokens, 30s |
| `script-coach` | Reviews YouTube script for spoken flow | ~10k tokens, 30s |
| `outline-validator` | Checks deck-outline.md is build-ready | ~5k tokens, 10s |

## Going from script to actual YouTube video

The `youtube.md` file is the script. Producing the video is a separate workflow.

**Recommended stack:**

- **Descript** (~$30/mo) — script-driven editing, drops audio + slides into a timeline
- **ElevenLabs Pro** ($22/mo) — clone your voice once, generate audio in your voice from any script in seconds
- **Your laptop webcam** — for the 60-second talking-head intro
- **QuickTime / Loom** — for B-roll screen recordings

**Per-video workflow (~30-60 min after script is ready):**

1. Export deck slides as PNGs (PowerPoint: File → Export → Change file type → PNG)
2. Record the 60-second intro on your webcam
3. Paste the script body into Descript
4. Generate AI voiceover in your cloned voice
5. Drop the intro MP4 at the start, drag slide PNGs onto the timeline where the script says `[show slide N]`
6. Capture B-roll for `[B-roll: ...]` markers (or skip for v1)
7. Auto-generate captions
8. Export 1080p MP4 → upload to YouTube

**Avoid:** Clipchamp (no script-driven workflow), HeyGen / Synthesia AI avatars (uncanny for technical content), hiring a video editor before 10 videos shipped.

## Troubleshooting

### `/start-topic` doesn't seem to handle the file path correctly

The skill is meant to copy your markdown file into the new topic folder. If it doesn't, fall back to a two-step manual setup:

```bash
mkdir -p topics/02-my-topic-slug
cp ~/Desktop/my-notes.md topics/02-my-topic-slug/notes.md
```

Then edit `topics/02-my-topic-slug/topic.yaml`:

```yaml
topic_number: 2
topic_slug: my-topic-slug
date_started: 2026-05-17
topic_name: "Your topic name"
short_description: "One line"
headline_concept: "The thing that gets purple"
angle: ""
slide_count: 18
status: in_progress
```

Then `/finish-topic` works normally.

### The build fails with "MODULE_NOT_FOUND: pptxgenjs"

Pptxgenjs isn't installed globally where the script can find it. Re-run:

```bash
npm install -g pptxgenjs
```

And confirm `npm root -g` outputs a path. The build script uses `NODE_PATH="$(npm root -g)"` to find the module.

### A slide shows text overflow or overlap

This means the content in `deck-outline.md` is too long for that layout. Two fixes:

1. **Best fix:** Edit `notes.md` to make the source content tighter, then `/finish-topic` again. The deck-architect will produce shorter copy.
2. **Quick fix:** Edit the specific slide block in `topics/<NN>-<slug>/deck-outline.md` to shorten the text, then `/build-deck` to re-render.

### The post sounds generic / not in my voice

`brand/linkedin-style.md` is the voice anchor. After your first 2-3 published posts, refine that file based on what you actually changed in the drafts. Specific phrases you keep deleting → add to the "Things to avoid" section. Specific phrases you keep adding → quote them in the "What this voice does" section.

The system gets sharper post by post as the anchor evolves.

### Subagent times out or returns nothing useful

Usually means the input is too sparse. The `deck-architect` needs at least 500-1000 chars of notes to design 18 slides. `angle-finder` needs specific content to propose specific angles. If notes are vague, all outputs will be vague.

## What's intentionally NOT here yet

These are deliberate omissions. Add when stable:

| Feature | Status | When to revisit |
|---|---|---|
| Publishing automation (LinkedIn API, YouTube upload) | Manual push only | After 5+ topics shipped |
| Video rendering with voice clone | Manual workflow in Descript | After Descript workflow is stable |
| Companion GitHub repo spawning for code-heavy topics | Manual | When a topic actually demands code |
| Channels beyond LinkedIn + YouTube | Excluded by design | Probably never — focus wins |
| Cowork integration for scheduling | Excluded | After publishing is stable |
| Memory of past posts to refine voice | Manual editing of linkedin-style.md | After 5+ posts |

Build the boring foundation first. Add automation only after you know what you're publishing.
